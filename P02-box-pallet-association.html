一、系统简介
1.1 系统组成

本系统采用四级关联模式（瓶-盒-箱-垛），由两个独立软件组成：

  • 1号机（瓶盒关联软件）：负责瓶码和盒码的采集关联
  • 2号机（盒箱垛关联软件）：负责盒箱垛关联采集、数据校验、数据管理和数据上传
1.2 包装规格

默认值，可在系统设置中配置，勿随意修改

层级关系
1盒= 6瓶
1箱= 4盒（24瓶）
1垛= 70箱（280盒，1680瓶）

3.2 数据采集主界面操作流程
生产前准备（开始采集前需完成）：
类别准备项说明
门禁码包已导入盖外码小标、盒外码中标、箱外码大标；未导入则提示，支持在线拉取或本地导入
必填产品已选择左侧「产品选择」下拉框选中产品
必填生产单号已填写默认当天日期+001，可修改
规格采集规格已确认1垛 [70]箱、1箱 [4]盒；开始采集后不可修改
设备设备连接正常盒码相机、箱码相机、剔除装置、报警灯等
连接数据库可访问系统设置→连接→数据库连接测试通过
  1. 在左侧「当前生产信息」区域，点击「产品选择」下拉框，在弹窗中搜索并选择生产产品
  2. 确认生产单号（系统自动生成，可手动修改）
  3. 确认采集规格（1垛 [70]箱、1箱 [4]盒，70、4可修改）；采集开始后不可修改
开始采集：
  1. 点击右侧「开始采集」按钮
  2. 系统检测设备状态，检测通过后弹出二次确认弹窗，展示生产信息（产品、生产单号）和包装比例（1垛X箱、1箱X盒）
  3. 确认后开始采集
监控采集过程：
  • 中间上方 - 数据接收区：扫码数据和关联结果，最新在上
  • 中间中部 - 操作日志：用户操作记录，最新在上
  • 中间下方 - 报警信息：重码、关联失败等异常，最新在上
  • 右侧 - 单位实时统计：当前箱数、每垛箱数、当前盒数、每箱盒数、已生产垛数、总剔除数，文字在上数字在下，每项独立卡片
成垛与自动上传：
  • 箱数达到预设值时自动成垛
  • 成垛后自动上传数据至云端（系统设置→业务→上传配置中「自动上传开关」开启时，默认开启；关闭时需手动上传）
  • 上传成功：绿灯常亮，无蜂鸣/提示音；一分钟后自动熄灭
  • 上传失败：红灯常亮 + 蜂鸣；报警信息区显示具体失败原因

四、各功能操作说明
4.1 手工采集（瓶盒采集相机损坏时使用，人工扫码补录瓶盒关联）
  1. 点击「手工采集」Tab
  2. 设置采集规格（1盒 [6]瓶，6可修改）；采集开始后不可修改

  6. 扫码完成后自动完成瓶盒关联，当前已读数量归零，继续下一组
4.2 码包管理

用途：导入包材厂提供的码包文件，用于采集时比对校验（不在码包内的码会被剔除）。

操作步骤
启动时在线导入软件启动时自动执行，仅拉取盖外码小标；盒外码中标、箱外码大标不参与
本地导入点击后弹窗 → 先选择码包类型（盖外码小标/盒外码中标/箱外码大标）→ 再选择 TXT 文件 → 输入密码 → 确认解析入库。盒外码中标、箱外码大标仅能本地导入；盖外码小标无网络时也可本地导入。因格式相同，类型需人工指定
在线更新点击「在线更新」→ 仅拉取盖外码小标（盒外码中标、箱外码大标不参与，均需本地导入）
查看码包点击操作列「查看」→ 弹窗显示该码包所有码，支持搜索
删除码包仅当该码包内无已关联码时允许删除；有则禁止并提示。确认后逻辑删除

码包格式说明：TXT 文件，每行一个码（盖外码小标=瓶码，盒外码中标=盒码，箱外码大标=箱码）

4.3 数据查询

用途：查询码的关联关系，用于产品追溯。

输入码后点击「查询」或回车，查出该码所属垛的全部数据。例如输入箱码，显示该垛下所有箱（箱1、箱2…）及各自的盒、瓶。左侧表格按层级分列展示，与输入的码对应的单元格红色字体显示；点击某行，右侧展示该行的具体信息（码、采集时间、产品编号、产品名称、生产单号）。

4.4 数据替换 需要密码

用途：码损坏或码错误时，将旧码替换为新码（仅操作外码：瓶码/盒码/箱码，无需操作内码）。

  1. 输入原码（待替换）
  2. 输入新码（替换后）
  3. 输入替换原因（可选）
  4. 点击「确认替换」→ 弹窗核对信息 → 输入密码 → 确认执行

新码要求（需同时满足）：

  • ✅ 与原码同层级（瓶码只能替换瓶码，盒码只能替换盒码，箱码只能替换箱码）
  • ✅ 在对应层级码包中，且从未使用过（系统中不存在该码的关联记录）
  • ✅ 格式有效，且不能与原码相同

替换执行方式（系统自动判断）：

  • 未上传（仅本地）：校验通过后直接替换本地数据
  • 已上传：先替换云端再替换本地，保证一致性；若码类型有内码（盖码、箱码），系统自动同步替换云端内码，无需手动操作

⚠ 替换操作不可恢复，请务必核对无误再执行

4.5 取消关联 需要密码

用途：取消码的关联关系（如数据错误、产品需要重新处理时）。

重要规则：
  • 不能跳过上级直接取消下级：若指定码有上级关联，系统在识别时标记为不可取消，须先将上级码加入列表处理
  • 已上传云端时，系统先取消云端（整垛为单位），成功后再取消本地，确保一致性
取消范围（执行前选择）：
  • 只解一层：垛码断箱-垛、箱码断盒-箱、盒码断瓶-盒，下一层保留
  • 全部解除：箱码断盒-箱及瓶-盒（垛码受限等同只解一层）
操作步骤：
  1. 输入盒/箱/垛码（支持扫瓶码，系统自动转换为盒码）→ 点击「加入列表」
  2. 可批量加入多个码到待取消列表
  3. 点击「识别」→ 右侧显示各码识别结果（✓ 可取消 / ✗ 不可取消并提示原因）
  4. 若有码标记为不可取消（有上级），将上级码也加入列表，再次点击「识别」
  5. 选择取消范围（只解一层 / 全部解除）
  6. 点击「确认取消关联」→ 输入密码 → 执行

⚠ 操作不可恢复，请确认无误后操作

4.6 生产统计

用途：查看生产数据统计和各垛上传状态。

操作步骤
生产统计查询左侧选择日期（开始-结束）、可输入生产单号 → 点击「查询」→ 显示垛数/箱数/盒数
查看垛码列表点击蓝色下划线的「垛数」数字 → 弹窗显示垛码列表，支持模糊筛选
上传统计查询右侧选择日期、生产单号、状态（全部/成功/异常）→ 点击「查询」
4.7 数据上传

用途：管理上传任务和状态，处理上传异常。

功能操作
手动上传点击「手动上传」→ 批量上传所有未上传的垛
查询状态输入垛码 → 点击「查询状态」→ 查看上传详情
设为未上传输入垛码 → 点击「设为未上传」→ 确认（用于异常修复后重新补传）
设为已上传输入垛码 → 点击「设为已上传」→ 确认（避免系统重复上传）

左侧「上传信息展示区」实时显示自动上传日志：

  • ■ 灰色：开始上传
  • ■ 蓝色：上传中
  • ■ 绿色：上传成功
  • ■ 红色：上传失败（含失败原因）
五、常见问题 & 异常处理
Q1：2号机提示码校验异常（重码/错码/超码/缺码），怎么处理？
2号机统一执行码包校验、重码检测和G数量校验。检测到异常后，该箱数据标记无效并触发剔除装置，产线正常移动，无需手动干预。查看报警信息区了解具体原因，根据提示处理实物后重新上线。
Q2：设备显示未连接怎么办？
  1. 检查读码器/相机的电源和网线是否插好
  2. 重启设备后，查看连接状态是否恢复
  3. 若仍未连接，联系技术人员
Q3：数据上传失败，红灯亮了怎么办？
  1. 查看「数据上传」Tab 中的失败原因
  2. 根据失败原因进行数据处理或等待网络恢复
  3. 点击「关闭报警」重置三色声光报警灯（停止灯光和蜂鸣）
  4. 处理完成后，点击「手动上传」重新上传
Q4：生产结束时还有尾数（未满垛），怎么处理？
方式一（推荐）：点击「强制满垛」，将当前数据强制成垛并上传
方式二：直接停止采集，退出软件前选择暂存数据。下次生产时点击「提取工单未成垛」，输入/扫描垛内任一箱码，查出对应生产订单并确认，回显主页面继续生产
Q4-1：退出软件时有未满垛数据，弹窗怎么选？
暂存：将未满垛数据存库标记未成垛，允许退出；下次可「提取工单未成垛」继续生产
强制满垛：执行强制满垛流程（生成虚拟垛标、重置计数），操作完毕后允许退出
取消：返回主界面，不退出
Q5：剔除装置剔出去后没有自动收回，怎么办？
点击右侧「收回剔除」按钮，手动触发收回指令。
Q6：忘记密码怎么办？
系统设置、码替换、取消关联均需输入密码（密码请联系管理员获取）
Q7：如何知道码属于哪一层级（瓶/盒/箱/垛）？
在「数据查询」Tab 中输入码值，点击「查询」或回车，左侧表格按层级分列展示，点击某行右侧可查看该行的详细信息。
Q8：识别结果提示「不可取消，有上级关联」，怎么操作？
  1. 在识别结果中找到标记 ✗ 的码，查看提示（如「已关联至垛码xxx，请先将垛码加入列表」）
  2. 将提示的上级码（垛码）也输入并「加入列表」
  3. 再次点击「识别」，上级码通过后该码变为 ✓ 可取消
  4. 确认无 ✗ 码后，点击「确认取消关联」→ 输入密码 → 执行
六、注意事项
  1. 密码保护：系统设置、码替换、取消关联均需要输入密码确认
  2. 不可恢复操作：码替换、取消关联执行后无法撤销，操作前请仔细核对
  3. 包装比例：不可随意修改，如需变更请联系管理员
  4. 生产前检查：每次开始生产前，确认码包已导入、设备连接正常
  5. 异常优先处理：2号机发现报警信息（中间下方红色区域）应及时处理，避免影响数据准确性
