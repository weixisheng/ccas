一、系统简介
1.1 系统组成

本系统采用四级关联模式（瓶-盒-箱-垛），由两个独立软件组成：

  • 1号机（瓶盒关联软件）：负责瓶码和盒码的采集关联，数据实时同步至2号机
  • 2号机（盒箱垛关联软件）：负责盒箱垛关联采集、数据管理和数据上传
1.2 包装规格

默认值，可在系统设置中配置，勿随意修改

层级关系
1盒= 6瓶
1箱= 4盒（24瓶）
1垛= 70箱（280盒，1680瓶）
二、1号机操作指南（瓶盒关联软件）
2.1 日常操作流程
步骤1：确认采集规格
  • 在左侧「采集规格设置」区域，确认「1盒 6瓶」设置正确（6可修改，默认6瓶）
  • ⚠ 开始采集后不可修改
步骤2：开始采集
  1. 点击「开始采集」按钮
  2. 系统自动检测设备状态（瓶1读码器、瓶2读码器、盒读码器）
  3. 检测通过后开始采集；按钮变为「停止采集」
步骤3：监控采集过程
  • 右侧 - 数据接收区：盒码、瓶码及关联结果统一展示，时间+具体信息每行一条；绿色=关联成功，红色=关联失败或异常；黑色=正常码，红色=重码/错码（系统会标记剔除，产线正常移动，下游按箱剔除）
  • 左侧 - 单位实时统计：「合计瓶数」「当前盒数」实时更新（文字在上、数字在下）
步骤4：停止采集
  • 点击「停止采集」按钮，未满盒的数据会保留
2.2 设备连接状态
  • 右侧操作日志区域显示读码器连接状态
  • 绿色文字：设备已连接 ✓
  • 红色文字：设备未连接，请检查设备连接后重启
2.3 清屏
  • 点击「清屏」按钮：清空数据接收区显示记录
  • ⚠ 清屏不影响计数和数据库数据

3.2 数据采集主界面操作流程
生产前准备：
  1. 在左侧「当前生产信息」区域，点击「产品选择」下拉框，搜索并选择生产产品
  2. 确认生产单号（系统自动生成，可手动修改）
  3. 确认采集规格（1垛 [70]箱、1箱 [4]盒，70、4可修改）；采集开始后不可修改
开始采集：
  1. 点击右侧「开始采集」按钮
  2. 系统检测设备状态，检测通过后开始采集
  3. 报警灯绿灯常亮，表示系统正常工作
监控采集过程：
  • 中间上方 - 数据接收区（约60%高）：扫码数据和关联结果，最新在上
  • 中间中部 - 操作日志（约20%）：用户操作记录，最新在上
  • 中间下方 - 报警信息（约20%）：重码、关联失败等异常，最新在上
  • 右侧 - 单位实时统计：当前箱数、每垛箱数、当前盒数、每箱盒数、已生产垛数、总剔除数，文字在上数字在下，每项独立卡片
成垛与自动上传：
  • 箱数达到预设值时自动成垛，成垛后自动上传数据至云端
  • 上传成功：绿灯常亮 + 声音提醒
  • 上传失败：红灯常亮 + 声音提醒（需处理后手动重传）
3.3 右侧任务控制

总剔除数显示在单位实时统计区域。按钮一行两个排列：

按钮功能说明
开始/停止采集启动或停止数据采集；停止后未满垛数据保留
无需采集码当天不生产五码合一产品时开启，所有读码设备不工作
关闭报警上传失败触发报警后，点击可重置报警状态（停止声光报警器、蜂鸣器）
强制满垛未达到预设箱数时，强制结束当前垛，生成垛码
提取工单未成垛弹窗输入/扫描垛内任一箱码，查出对应生产订单并显示，确认后回显主页面继续生产
收回剔除剔除设备动作后未自动收回时，点击手动收回
剔除数清零将总剔除数重置为 0
四、各功能操作说明
4.1 手工采集（1号机相机损坏时使用，仅支持瓶盒关联）
  1. 点击「手工采集」Tab
  2. 设置采集规格（1盒 [6]瓶，6可修改）；采集开始后不可修改
  3. 点击「开始采集」按钮，按钮变为「停止采集」
  4. 按左侧「开始采集提示」顺序用扫码枪扫码：先扫瓶码（足够数量后）→ 扫盒码
  5. 左侧开始采集提示区文字随进度变化，告知当前应扫瓶码或盒码（无指示灯，文字提示即可）
  6. 扫码完成后自动完成瓶盒关联，当前已读数量归零，继续下一组
4.2 码包管理

用途：导入包材厂提供的码包文件，用于采集时比对校验（不在码包内的码会被剔除）。支持的码包类型：盖外码小标、盒外码中标、箱外码大标。

操作步骤
启动时在线导入软件启动时自动执行，同名码包跳过不重复导入
本地导入点击「本地导入」→ 先选择码包类型（盖外码小标/盒外码中标/箱外码大标）→ 选择 TXT 文件 → 确认导入。因文件格式相同，系统无法自动识别类型，需人工指定
在线更新点击「在线更新」→ 从在线源拉取码包（同名跳过），分别拉取三种类型
查看码包点击操作列「查看」→ 弹窗显示该码包所有码，支持搜索与分页
删除码包仅当该码包内无任何已关联码时允许删除；若有已关联码则禁止删除并提示。删除为逻辑删除

帮助提示：点击顶部操作区「?」图标可查看详细的码包导入说明、格式说明及删除规则。

4.3 数据查询

用途：查询码的关联关系，用于产品追溯。

输入码后点击「查询」或回车，查出该码所属垛的全部数据。例如输入箱码，显示该垛下所有箱（箱1、箱2…）及各自的盒、瓶。左侧表格按层级分列展示，与输入的码对应的单元格红色字体显示；点击某行，右侧展示该行的具体信息（码、采集时间、产品编号、产品名称、生产单号）。

4.4 数据替换 需要密码 123456

用途：码损坏或码错误时，将旧码替换为新码。

  1. 输入原码（待替换）
  2. 输入新码（替换后）
  3. 输入替换原因（可选）
  4. 点击「确认替换」→ 弹窗核对信息 → 输入密码 123456 → 确认执行

新码要求（需同时满足）：

  • ✅ 在导入的码包范围内
  • ✅ 未被使用过（系统中无关联关系）
  • ✅ 格式有效，且不能与原码相同

⚠ 替换操作不可恢复，请务必核对无误再执行

4.5 取消关联 需要密码 123456

用途：解除码的关联关系（如数据错误、产品需要重新处理时）。

重要规则：
  • 必须从大到小逐级解除（垛 → 箱 → 盒 → 瓶），不可一次性解除全部
  • 系统自动检查上级关联状态，有上级时「确认」按钮禁用，提示先解除上级
  • 系统自动检查云端数据并先解除云端关联，确保数据一致
单个取消关联操作步骤：
  1. 选择「单个取消关联」模式
  2. 输入同一包装内的任意两个码
  3. 点击「识别」→ 右侧显示识别结果（含上级关联链路）
  4. 如有上级关联，先从垛开始逐级解除上级，再回来解除当前级
  5. 无上级关联后，点击「确认取消关联」→ 输入密码 123456 → 确认执行
批量取消关联操作步骤：
  1. 选择「批量取消关联」模式
  2. 逐一输入大码（盒码/箱码/垛码）→ 点击「添加」
  3. 点击「识别」→ 右侧显示所有包装单元的识别结果
  4. 处理有上级关联的单元后，点击「确认批量取消关联」→ 输入密码 123456 → 执行

⚠ 操作不可恢复，请确认无误后操作

4.6 生产统计

用途：查看生产数据统计和各垛上传状态。

操作步骤
生产统计查询左侧选择日期范围 → 点击「查询」→ 显示垛数/箱数/盒数/剔除数
查看垛码列表点击「垛数」数字 → 弹窗显示垛码列表（垛码、箱数、关联时间），支持按垛码筛选
查看剔除记录点击「剔除数」数字 → 弹窗显示剔除记录，左右分栏（分层表格+详情），支持按箱码筛选
上传统计查询右侧选择日期范围和状态（全部/成功/异常）→ 点击「查询」
4.7 数据上传

用途：管理上传任务和状态，处理上传异常。

功能操作
手动上传点击「手动上传」→ 批量上传所有未上传的垛
查询状态输入垛码 → 点击「查询状态」→ 查看上传详情
设为未上传输入垛码 → 点击「设为未上传」→ 确认（用于异常修复后重新补传）
设为已上传输入垛码 → 点击「设为已上传」→ 确认（避免系统重复上传）

左侧「上传信息展示区」实时显示自动上传日志：

  • ■ 灰色：开始上传
  • ■ 蓝色：上传中
  • ■ 绿色：上传成功
  • ■ 红色：上传失败（含失败原因）
五、常见问题 & 异常处理
Q1：1号机显示红色码，怎么处理？
红色码 = 重码（已存在）或错码（格式错误）。系统自动标记待剔除，产线正常移动，下游龙门架按箱剔除，无需手动干预。
Q2：设备显示未连接怎么办？
  1. 检查读码器/相机的电源和网线是否插好
  2. 重启设备后，查看连接状态是否恢复绿色
  3. 若仍未连接，联系技术人员
Q3：数据上传失败，红灯亮了怎么办？
  1. 查看「数据上传」Tab 中的失败原因
  2. 根据失败原因进行数据处理或等待网络恢复
  3. 点击「关闭报警」重置报警灯和蜂鸣器
  4. 处理完成后，点击「手动上传」重新上传
Q4：生产结束时还有尾数（未满垛），怎么处理？
方式一（推荐）：点击「强制满垛」，将当前数据强制成垛并上传
方式二：直接停止采集，不做处理。下次生产时点击「提取工单未成垛」，选择该条数据继续生产
Q5：剔除装置剔出去后没有自动收回，怎么办？
点击右侧「收回剔除」按钮，手动触发收回指令。
Q6：忘记密码怎么办？
